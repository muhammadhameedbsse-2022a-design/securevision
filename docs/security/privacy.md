# SecureVision Privacy and Security Policy

SecureVision processes biometric data through face recognition technology.  
To protect user privacy and maintain security, the following policies are implemented.

## Biometric Data Handling
- Facial data is processed only for security and identification purposes.
- Raw images are not permanently stored unless required for authorized profiles.
- Face embeddings are used instead of raw images whenever possible.

## User Consent
- Users must provide explicit consent before facial data is captured.
- Clear notifications will inform users when camera monitoring is active.
- Users have the right to request deletion of their biometric data.

## Encryption
- All sensitive data is encrypted during transmission using HTTPS.
- Stored biometric embeddings are encrypted at rest.
- Secure key management practices are implemented.

## Audit Logs
- System events are recorded in secure audit logs.
- Logs track authentication attempts, model detections, and system access.
- Logs help identify misuse or unauthorized access.

## Storage Policies
- Biometric data is stored only when necessary for recognition features.
- Data storage follows secure database practices.
- Access is restricted to authorized system components.

## Retention Limits
- Facial data and logs are retained only for a limited time.
- Old detection logs are automatically deleted after the retention period.
- Retention policies comply with privacy regulations where applicable.
